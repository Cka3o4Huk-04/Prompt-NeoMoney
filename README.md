# Проект: ИИ-ассистент по онбордингу NeoMoney

## 1. Обзор
**Ассистент по онбордингу NeoMoney** — это интеллектуальный цифровой гид, созданный для оптимизации процесса адаптации новых сотрудников в NeoMoney (современном цифровом необанке). Ассистент берет на себя рутинные вопросы по безопасности, корпоративным политикам, инструментам и чек-листам адаптации, снижая нагрузку на HR и IT-департаменты.

## 2. Техническая архитектура (GitOps подход)
Для обеспечения максимальной гибкости и масштабируемости проект построен на разделенной архитектуре:

- **Ядро системы (Python):** Универсальное приложение, которое управляет взаимодействием с LLM и реализует RAG (Retrieval-Augmented Generation — генерация с дополненным поиском).
- **Слой конфигурации (GitHub):** Все «знания» и правила поведения бота хранятся в GitHub-репозитории. Это позволяет обновлять логику бота без необходимости переразвертывания кода на сервере.
  - `config/config.json`: Глобальные настройки, параметры модели и контакты для эскалации.
  - `prompts/system_prompt.txt`: Личность бота, правила и операционная логика.
  - `data/knowledge_base.csv`: Структурированный «источник истины» со всей корпоративной информацией.

### Поток данных:
`Запрос пользователя` $\rightarrow$ `Python-движок` $\rightarrow$ `Загрузка Конфига/БЗ из GitHub` $\rightarrow$ `LLM (с Системным промптом)` $\rightarrow$ `Структурированный ответ`

## 3. Компоненты проекта

### 📂 Конфигурация (`/config`)
Содержит `config.json`, в котором определены:
- Настройки модели (GPT-4o, температура 0.3).
- Точки эскалации (email-адреса HR, IT-поддержки и руководителей).
- Интервалы синхронизации с GitHub.

### 📂 База знаний (`/data`)
Структурированный CSV-файл (`knowledge_base.csv`), содержащий:
- **Категории**: Безопасность, Организация, Инструменты, Онбординг.
- **Ключевые слова**: Для эффективного поиска информации.
- **Маппинг эскалации**: Прямые ссылки на нужный отдел, если ИИ не может решить проблему самостоятельно.

### 📂 Промпты (`/prompts`)
Файл `system_prompt.txt` определяет личность «NeoMoney»:
- **Роль**: Профессиональный, дружелюбный и надежный помощник.
- **Правила**: Строгое следование Базе Знаний, отсутствие галлюцинаций и четкие границы в вопросах кадров и юриспруденции.

### 📂 Тестирование (`/tests`)
Комплексный набор тестовых сценариев (`test_scenarios.csv`), включающий:
- Позитивные кейсы (стандартные вопросы).
- Граничные случаи (запросы о зарплате).
- Запросы вне области компетенции.
- Триггеры эскалации.

### 📂 Примеры диалогов (`/examples`)
Файл `dialogues.md` содержит полноценные примеры диалогов «вопрос новичка → ответ ассистента → эскалация», демонстрирующие работу системного промпта и логики эскалации в реальных сценариях.

## 4. Быстрый старт для разработчика (Python Implementation)
Для реализации данного проекта на Python рекомендуется следующий стек:
- **Библиотеки**: `pandas` (для работы с CSV), `openai` (для доступа к LLM), `requests` (для синхронизации с GitHub API).
- **Логика поиска**: 
  1. Извлечь ключевые слова из запроса пользователя.
  2. Найти подходящую строку в `knowledge_base.csv` по колонке `keywords`.
  3. Передать найденный `answer` и `system_prompt.txt` в LLM.
  4. Если в строке указан `escalation`, добавить в ответ контакт из `config.json`.

## 5. Развертывание и поддержка
1. **Развертывание**: Python-движок разворачивается на сервере (например, через Docker/AWS).
2. **Обновление знаний**: Чтобы добавить новое правило или обновить процесс, достаточно отредактировать `knowledge_base.csv` в GitHub и закоммитить изменения.
3. **Обновление личности**: Изменение `system_prompt.txt` позволяет мгновенно поменять тон общения или добавить новые операционные правила.

## 6. Заключение
Данная архитектура превращает обычного чат-бота в профессиональный корпоративный инструмент. Разделение логики и данных позволяет NeoMoney масштабировать процесс онбординга на разные отделы и страны, просто добавляя новые CSV-файлы и обновляя системный промпт.
