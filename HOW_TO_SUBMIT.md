# Инструкция: Как перенести проект в Google Docs

Данная инструкция описывает, как подготовить и сдать проект «ИИ-ассистент по онбордингу NeoMoney» в Google Docs, сохранив форматирование Markdown (заголовки, таблицы, списки, код).

---

## Способ 1: Рекомендуемый (через Markdown Preview + Копирование)

Этот способ сохраняет максимальное форматирование (таблицы, заголовки, жирный шрифт, списки).

### Шаг 1: Откройте файл `PROJECT_SUBMISSION.md`
В VS Code откройте файл `PROJECT_SUBMISSION.md` (он находится в корне проекта).

### Шаг 2: Включите предпросмотр Markdown
- Нажмите сочетание клавиш `Ctrl + Shift + V` (Windows/Linux) или `Cmd + Shift + V` (Mac).
- Либо кликните правой кнопкой мыши по вкладке файла и выберите **「Open Preview」**.

### Шаг 3: Выделите весь контент
В окне предпросмотра (справа) нажмите `Ctrl + A` (Windows/Linux) или `Cmd + A` (Mac), чтобы выделить весь текст.

### Шаг 4: Скопируйте
Нажмите `Ctrl + C` / `Cmd + C`.

### Шаг 5: Вставьте в Google Docs
1. Откройте [Google Docs](https://docs.google.com) и создайте новый документ.
2. Нажмите `Ctrl + V` / `Cmd + V`.
3. **Важно:** Google Docs автоматически преобразует заголовки (`#`, `##`), жирный текст (`**текст**`), списки и таблицы в нативное форматирование Docs.

### Шаг 6: Проверьте форматирование
Пройдитесь по документу и убедитесь, что:
- Заголовки отображаются как «Заголовок 1», «Заголовок 2» (меню `Формат` → `Абзацные стили`).
- Таблицы (База знаний, Тестовые сценарии) корректно отображаются.
- Блоки кода (Системный промпт, Конфигурация) выделены моноширинным шрифтом.

## Способ 2: Через онлайн-конвертер (Markdown → Google Docs)

Если первый способ дал сбои с таблицами, используйте конвертер.

### Шаг 1: Скопируйте содержимое файла
Откройте `PROJECT_SUBMISSION.md` в VS Code, выделите всё (`Ctrl + A`) и скопируйте (`Ctrl + C`).

### Шаг 2: Используйте конвертер
1. Перейдите на сайт [https://www.vertopal.com/ru/convert/md-to-docx](https://www.vertopal.com/ru/convert/md-to-docx) или аналогичный.
2. Вставьте скопированный Markdown-текст.
3. Нажмите **Конвертировать** и скачайте файл `.docx`.

### Шаг 3: Загрузите в Google Docs
1. В Google Drive нажмите **「Создать」** → **「Загрузить файл」**.
2. Выберите скачанный `.docx` файл.
3. Google Docs автоматически откроет его в редактируемом формате.

## Способ 3: Ручное копирование по разделам (если документ большой)

Если единый документ получается слишком громоздким, разбейте его на логические части:

| Раздел | Что копировать | Куда вставлять |
|--------|----------------|----------------|
| **Титульная страница** | Название проекта, автор, курс, дата | Первый документ Google Docs |
| **Системный промпт** | Содержимое `prompts/system_prompt.txt` | Отдельный документ «Системный промпт» |
| **База знаний** | Таблицу из `data/knowledge_base.csv` | Google Таблица (Google Sheets) |
| **Примеры диалогов** | Содержимое `examples/dialogues.md` | Отдельный документ «Примеры диалогов» |
| **Тесты** | Таблицу из `tests/test_scenarios.csv` | Google Таблица (Google Sheets) |

> **Совет:** Для таблиц (База знаний, Тесты) лучше использовать **Google Sheets**, а в основном документе Google Docs делать на них гиперссылки.

## Чек-лист перед сдачей

Перед тем как отправить ссылку на документ преподавателю, проверьте:

- [ ] Все заголовки оформлены стилями (Заголовок 1, Заголовок 2).
- [ ] Таблицы читаемы и не «поехали».
- [ ] Блоки кода выделены моноширинным шрифтом (`Courier New` или `Roboto Mono`).
- [ ] Нет «битых» символов Markdown (например, `##` или `**` в виде текста).
- [ ] Документ открыт по ссылке для просмотра/комментирования (Настройки доступа → «Все, у кого есть ссылка»).
